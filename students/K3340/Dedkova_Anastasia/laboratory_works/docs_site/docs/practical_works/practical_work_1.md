# Практическая работа 1.1

## Тема
Разработка серверного приложения FastAPI по теме буккроссинга.

## Цель работы
Освоить создание базового серверного приложения на FastAPI, настройку маршрутов, использование Pydantic-моделей и реализацию CRUD-операций на временных данных.

## Что было реализовано
Создано API на FastAPI по теме буккроссинга.

Реализованы сущности:
- книги;
- профили пользователей;
- жанры.

Для сущности книги добавлены вложенные объекты:
- владелец книги;
- список жанров.

## Реализованные маршруты

### Главный маршрут
- `GET /`

### Книги
- `GET /books/`
- `GET /books/{book_id}`
- `POST /books/`
- `DELETE /books/{book_id}`

### Профили
- `GET /profiles/`
- `GET /profiles/{profile_id}`
- `POST /profiles/`
- `DELETE /profiles/{profile_id}`

### Жанры
- `GET /genres/`
- `GET /genres/{genre_id}`
- `POST /genres/`
- `DELETE /genres/{genre_id}`

## Использованные технологии
- Python
- FastAPI
- Uvicorn
- Pydantic


## Запуск проекта
` uvicorn app.main:app --reload `

## Результат

Создано базовое серверное приложение FastAPI с временным хранилищем данных и автоматически сгенерированной документацией Swagger.
