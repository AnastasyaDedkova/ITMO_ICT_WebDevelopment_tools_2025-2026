# Практическая работа 1.3

## Тема
Использование Alembic для управления миграциями базы данных в FastAPI-проекте.

## Цель работы
Научиться настраивать Alembic, создавать миграции и применять изменения схемы базы данных в проекте на FastAPI с PostgreSQL.

## Что было сделано
В ходе практической работы был подключён и настроен Alembic для проекта BookCrossing API.

Выполнены следующие шаги:
1. Установлена библиотека Alembic.
2. Выполнена инициализация Alembic в проекте.
3. Созданы файлы `alembic.ini` и директория `migrations`.
4. Настроен файл `migrations/env.py` для работы с SQLModel и переменной окружения `DATABASE_URL`.
5. Сгенерирована начальная миграция по существующим моделям.
6. Начальная миграция применена к базе данных.
7. Автоматическое создание таблиц через `create_all` было убрано из приложения.
8. Выполнена дополнительная миграция после изменения модели.

## Результат
В результате практической работы в проекте была настроена полноценная система миграций базы данных. Это позволяет управлять изменениями структуры БД корректно и использовать их в дальнейшей разработке лабораторной работы.
